# Проверка гипотез в сервисе аренды самокатов Go-Fast

## Цель проекта  
Проанализировать демографию пользователей и особенности использования самокатов, а также определить возможную выгоду от распространения платной подписки на самокаты.   

# Описание данных

Для анализа поступили данные сервиса аренды самокатов «Go». Данные состоят из трёх датасетов:

- `users_go.csv` — информация о пользователях сервиса
- `rides_go.csv` — данные о поездках
- `subscriptions_go.csv` — информация о тарифных планах

### Описание датасета `users_go.csv`

| Поле | Тип | Описание |
|------|-----|----------|
| `user_id` | int | Уникальный идентификатор пользователя |
| `name` | string | Имя пользователя |
| `age` | int | Возраст |
| `city` | string | Город проживания |
| `subscription_type` | string | Тип подписки: `free` или `ultra` |

### Описание датасета `rides_go.csv`

| Поле | Тип | Описание |
|------|-----|----------|
| `user_id` | int | Идентификатор пользователя |
| `distance` | int | Расстояние поездки (метры) |
| `duration` | int | Продолжительность поездки (минуты) |
| `date` | date | Дата поездки |

### Описание датасета `subscriptions_go.csv`

| Поле | Тип | Описание |
|------|-----|----------|
| `subscription_type` | string | Тип подписки: `free` или `ultra` |
| `minute_price` | decimal | Стоимость минуты поездки (руб.) |
| `start_ride_price` | decimal | Стоимость начала поездки (руб.) |
| `subscription_fee` | decimal | Ежемесячный платёж (руб.) |

## Структура проекта

1. Загрузка данных и знакомство с ними.
2. Предобработка данных.
3. Исследовательский анализ данных.
   Опишите и визуализируйте общую информацию о пользователях и поездках:
   - частота встречаемости городов;
   - соотношение пользователей с подпиской и без подписки;
   - возраст пользователей;
   - расстояние, которое пользователь преодолел за одну поездку;
   - продолжительность поездок.
4. Объединение данных:
   - c данными о пользователях без подписки;
   - с данными о пользователях с подпиской.
5. Подсчёт выручки.
6. Проверка гипотез:
   - 6.1 Важно понять, тратят ли пользователи с подпиской больше времени на поездки? Если да, то пользователи с подпиской     могут быть «выгоднее» для компании. Проверьте гипотезу. Используйте данные о продолжительности каждой сессии — отдельно для подписчиков и тех, у кого нет подписки;
   - 6.2 Расстояние одной поездки в 3130 метров — оптимальное с точки зрения износа самоката. Можно ли сказать, что расстояние, которое проезжают пользователи с подпиской за одну поездку, не превышает 3130 метров? Проверить гипотезу и сделать вывод;
   - 6.3. Проверьте гипотезу о том, будет ли помесячная выручка от пользователей с подпиской по месяцам выше, чем выручка от пользователей без подписки. Сделать вывод.
7. Распределения
   - 7.1 Рассчитать среднюю длительность поездки и стандартное отклонение длительности на основе выборки;
   - 7.2 Оценка потенциала промоакции для длительных поездок (>30 мин);
   - 7.3 Исследование альтернативного сегмента (20-30 минут);
   - 7.4 Определение критической дистанции для защиты оборудования.
